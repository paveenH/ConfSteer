# ConfSteer Label Format

## Overview

Labels are generated by `make_labels.py` by comparing model answers before and after RSN-based activation steering (alpha ∈ {-4, +4}).

Each label file corresponds to one task file under `orig/`, saved to `labels/{model}/{task}/labels_{orig_filename}`.

---

## Label Fields

Each sample contains:

| Field | Type | Description |
|---|---|---|
| `index` | int | Row index — directly maps to the corresponding hidden state in the `.h5` file |
| `text` | str | Question + choices (used for alignment verification) |
| `true_label` | str / set | Correct answer letter(s). Set for TQA MC2, string for all others |
| `answer_orig` | str | Model's answer without steering (neutral role) |
| `answer_pos4` | str | Model's answer after +4 steering |
| `answer_neg4` | str | Model's answer after -4 steering |
| `orig_correct` | bool | Whether the original answer was correct |
| `label_pos4` | int | Steering outcome under +4 alpha (see encoding below) |
| `label_neg4` | int | Steering outcome under -4 alpha (see encoding below) |

---

## Label Encoding

```
+4 steering (confidence boost):
  +1  = worked    — wrong → correct
  -1  = backfire  — correct → wrong
   0  = no change

-4 steering (confidence suppression):
  -1  = worked    — correct → wrong
  +1  = backfire  — wrong → correct
   0  = no change
```

---

## Hidden State Alignment

Hidden states are stored in `.h5` files under `hidden_states_non/{model}/{task}/`.

Alignment is by **index**: `sample["index"]` in the label file corresponds directly to row `index` in the `.h5` dataset — no text matching needed.

```python
import h5py, json

with open("labels/llama3/mmlupro/labels_astronomy_8B_answers.json") as f:
    labels = json.load(f)["data"]

with h5py.File("hidden_states_non/llama3/mmlupro/neutral_astronomy_8B.h5", "r") as f:
    hidden_states = f["hidden_states"]  # shape: (N, layers, hidden_dim)

sample = labels[42]
h = hidden_states[sample["index"]]   # shape: (layers, hidden_dim)
y = sample["label_pos4"]             # -1, 0, or +1
```

## Class Distribution

Across all tasks (22,787 samples per model):

| Class | llama3 | qwen3 |
|---|---|---|
| +4 worked (+1) | 5.7% | 6.3% |
| +4 backfire (-1) | 5.8% | 4.5% |
| no change (0) | 88.5% | 89.2% |
| -4 worked (-1) | 4.0% | 7.8% |
| -4 backfire (+1) | 3.3% | 2.6% |

**Note:** `no_change` samples (≈89%) should be heavily downsampled during classifier training. Only a small fraction is needed to represent the baseline.
